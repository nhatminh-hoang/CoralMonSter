# PromptFreeMaskDecoder Explanation

This document explains the working mechanism of the `PromptFreeMaskDecoder` class in `CoralMonSter`, detailing the data flow and tensor shapes at each step.

## Overview

`PromptFreeMaskDecoder` is a wrapper around SAM's original `MaskDecoder`. It adapts the prompt-based decoding mechanism to a prompt-free, class-aware setting. Instead of user-provided points or boxes, it uses learnable **class tokens** to query the image embeddings and generate masks for specific semantic classes.

## Key Components

1.  **Class Tokens**: Learnable embeddings representing each semantic class.
2.  **Transformer**: A Two-Way Transformer that attends to both image embeddings and query (class) embeddings.
3.  **Upscaling Head**: A small CNN that upscales the processed image embeddings.
4.  **Hypernetworks**: MLPs that generate dynamic weights for mask prediction based on class token outputs.

## Process Flow & Tensor Shapes

We assume the following dimensions for this explanation:
*   $B$: Batch size
*   $C$: Transformer dimension (typically 256)
*   $H, W$: Image embedding spatial dimensions (typically $64 \times 64$)
*   $N$: Number of classes (`num_classes`)
*   $N_p$: Number of prompt embeddings (if any)

### 1. Input Preparation

The `forward` method takes:
*   `image_embeddings`: $(B, C, H, W)$
*   `image_pe` (Positional Encoding): $(1, C, H, W)$ or $(B, C, H, W)$
*   `prompt_embeddings` (Optional): $(B, N_p, C)$

**Step 1.1: Expand Positional Encoding**
*   `image_pe` is expanded to match batch size.
*   **Shape**: $(B, C, H, W)$

**Step 1.2: Prepare Queries**
*   `self.class_tokens` of shape $(N, C)$ are expanded to $(B, N, C)$.
*   If `prepend_class_queries` is True (default):
    *   `class_queries` are concatenated with `prompt_embeddings` (if provided).
    *   **Queries Shape**: $(B, N + N_p, C)$ (assuming $N_p=0$ for pure prompt-free, shape is $(B, N, C)$).
*   Let $N_{total} = N + N_p$.

### 2. Two-Way Transformer

The transformer processes the image and queries to capture geometric and semantic information.

*   **Input**:
    *   `image_embedding`: Flattened to $(B, HW, C)$.
    *   `point_embedding` (Queries): $(B, N_{total}, C)$.
*   **Process**:
    *   Self-attention on queries.
    *   Cross-attention: Queries attend to Image.
    *   MLP on queries.
    *   Cross-attention: Image attends to Queries.
*   **Output**:
    *   `hs` (Processed Queries): $(B, N_{total}, C)$
    *   `src` (Processed Image Embeddings): $(B, HW, C)$

### 3. Upscaling Image Embeddings

The processed image embeddings (`src`) are reshaped and upscaled to produce a high-resolution feature map for mask generation.

*   **Reshape**: `src` $\to$ $(B, C, H, W)$.
*   **Upscaling**:
    *   `self.output_upscaling` consists of two transposed convolutions (stride 2) separated by LayerNorm and Activation.
    *   Total upscaling factor: $4\times$.
    *   Output channels: $C/8$ (typically 32).
*   **Output (`upscaled`)**: $(B, C/8, 4H, 4W)$.
*   **Flatten**: `upscaled` is flattened spatially.
    *   **Output (`upscaled_flat`)**: $(B, C/8, 16HW)$.

### 4. Hypernetworks & Mask Generation

Masks are generated by computing the dot product between class-specific dynamic weights (from hypernetworks) and the upscaled image features.

**Step 4.1: Extract Class Tokens**
*   We take the last $N$ tokens from `hs` (corresponding to the class queries).
*   `token_block` = `hs[:, -N:, :]`
*   **Shape**: $(B, N, C)$.

**Step 4.2: Generate Dynamic Weights (Hypernetworks)**
*   `self.hypernets` is a list of $N$ MLPs (one per class).
*   Each MLP maps a class token $(C)$ to a weight vector $(C/8)$.
*   These are stacked to form `hyper_in`.
*   **Shape**: $(B, N, C/8)$.

**Step 4.3: Compute Mask Logits**
*   Matrix multiplication: `hyper_in` @ `upscaled_flat`.
    *   $(B, N, C/8) \times (B, C/8, 16HW) \to (B, N, 16HW)$.
*   Reshape to spatial dimensions.
*   **Output (`mask_logits`)**: $(B, N, 4H, 4W)$.
    *   These are the raw logits for the segmentation masks for each class.

### 5. Semantic Predictions (Auxiliary)

The decoder also produces semantic logits directly from the tokens, likely for classification or auxiliary supervision.

*   `semantic_tokens` = `token_block` $(B, N, C)$.
*   `self.semantic_head`: LayerNorm + Linear($C, N$).
*   **Output (`token_logits`)**: $(B, N, N)$.
    *   Note: This shape implies each of the $N$ class tokens predicts scores for all $N$ classes.

## Summary of Outputs

The `forward` method returns a dictionary:

1.  **`mask_logits`**: $(B, N, 4H, 4W)$ - The predicted segmentation masks.
2.  **`token_embeddings`**: $(B, N, C)$ - The refined class token embeddings.
3.  **`token_logits`**: $(B, N, N)$ - Classification scores derived from tokens.

## Loss Computation

The training of `CoralMonSter` involves a composite loss function that combines segmentation supervision with knowledge distillation from a teacher model. The loss components are weighted by the `DistillationConfig` arguments.

### 1. Segmentation Loss
This is the primary supervision signal, computed between the student's predicted masks and the ground truth masks.
*   **Components**:
    *   **Cross-Entropy Loss**: Standard pixel-wise classification loss.
    *   **Dice Loss**: Measures overlap between predicted and ground truth masks.
*   **Configuration**:
    *   `dice_weight`: Weight for Dice loss (default: 1.5).
    *   **`ce_weight`**: Weight for Cross-Entropy loss (default: 0.0).
*   **Formula**: $L_{seg} = w_{dice} \cdot L_{dice} + w_{ce} \cdot L_{ce}$

### 2. Mask Distillation Loss
Enforces consistency between the student's predicted masks and the teacher's predicted masks.
*   **Method**: Mean Squared Error (MSE) between student and teacher probability maps.
*   **Configuration**:
    *   `mask_kd_weight`: Weight for mask distillation (default: 0.0).
*   **Formula**: $L_{mask\_kd} = w_{mask\_kd} \cdot \text{MSE}(\sigma(Student), \sigma(Teacher))$

### 3. Token Distillation Loss
Aligns the student's class tokens with the teacher's class tokens in the embedding space.
*   **Method**: Can be either Cross-Entropy or KL Divergence.
    *   **Cross-Entropy**: Softmax on teacher tokens (soft targets) vs Log-Softmax on student tokens.
    *   **KL Divergence**: KL Divergence between student and teacher distributions.
*   **Configuration**:
    *   `token_kd_weight`: Weight for token distillation (default: 1.0).
    *   `token_kd_metric`: Metric to use ("ce" or "kl").
    *   `student_temperature`: Temperature scaling for student logits (default: 0.1).
    *   `teacher_temperature`: Temperature scaling for teacher logits (dynamic).
*   **Formula**: $L_{token\_kd} = w_{token\_kd} \cdot \text{Distill}(Student, Teacher)$

### 4. Token Classification Loss
Ensures that each learnable class token correctly predicts its corresponding class identity.
*   **Method**: Cross-Entropy loss with label smoothing.
*   **Input**: `token_logits` of shape $(B, N, N)$.
*   **Target**: Class indices $[0, 1, \dots, N-1]$.
*   **Configuration**:
    *   `token_classification_weight`: Weight for this auxiliary loss (default: 1.0).
*   **Formula**: $L_{token\_cls} = w_{token\_cls} \cdot \text{CE}(token\_logits, targets)$

### Total Loss
The final loss is the sum of all active components:
$$L_{total} = L_{seg} + L_{mask\_kd} + L_{token\_kd} + L_{token\_cls}$$
